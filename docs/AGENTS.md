프로젝트:
WWW Family Scheduler

이 프로젝트는:
- 가족 상태보드
- 생활 흐름 관리
- soft reminder
- lightweight family communication

앱이다.

절대:
- ERP
- 관리자툴
- audit log
- giant dashboard
- giant settings panel

처럼 보이면 안 된다.

핵심 디자인 철학:

1. 한 화면의 메인 hierarchy는 하나만
2. giant form 금지
3. card 안 card 금지
4. 설정 패널 느낌 금지
5. 생활 언어 사용
6. compact / soft / Toss 스타일 유지
7. avatar는 보조 identity 역할만
8. 상태보드 흐름 유지
9. 기능 추가보다 정리 우선
10. divider/spacing 기반 구조 우선

앱 구조:
- 홈 = 현재 상태
- 달력 = 탐색
- 보관함 = 기록
- 설정 = profile summary

항상:
- lightweight
- calm hierarchy
- low saturation
- compact spacing

유지.

금지:
- hard outline
- neon color
- giant CTA
- giant FAB
- giant avatar grid
- giant modal
- 관리자 용어
